# 打卡签到系统 - 需求文档

## 1. 项目概述

### 1.1 项目名称
基于位置的打卡签到系统（CheckIn System）

### 1.2 项目背景
为企业提供员工考勤管理解决方案，通过 GPS 定位技术实现精准打卡，防止代打卡行为，提高考勤管理效率。

### 1.3 目标用户
- 企业员工（打卡用户）
- 企业管理员（考勤管理人员）
- 系统管理员

---

## 2. 功能需求

### 2.1 用户模块

#### 2.1.1 用户注册/登录
- 支持手机号注册
- 支持账号密码登录
- 支持短信验证码登录
- 支持记住登录状态

#### 2.1.2 个人信息管理
- 查看/编辑个人资料
- 修改密码
- 上传头像

### 2.2 打卡模块

#### 2.2.1 上班打卡
- 获取当前 GPS 位置
- 判断是否在有效打卡范围内
- 记录打卡时间、位置信息
- 显示打卡结果（成功/失败及原因）

#### 2.2.2 下班打卡
- 同上班打卡逻辑
- 自动关联上午打卡记录

#### 2.2.3 打卡记录查询
- 按日期范围查询
- 查看打卡详情（时间、位置、状态）
- 导出打卡记录

### 2.3 考勤规则模块

#### 2.3.1 打卡点管理
- 创建打卡点（名称、地址、经纬度、有效半径）
- 编辑/删除打卡点
- 设置打卡点工作时间

#### 2.3.2 考勤规则配置
- 设置上班时间、下班时间
- 设置迟到/早退容忍时间
- 设置工作日配置

### 2.4 统计模块

#### 2.4.1 个人统计
- 月度出勤天数
- 迟到/早退/缺勤次数
- 考勤异常统计

#### 2.4.2 管理统计
- 部门考勤汇总
- 异常考勤报表
- 考勤数据导出

---

## 3. 非功能需求

### 3.1 性能要求
- 打卡接口响应时间 < 500ms
- 支持并发用户数 ≥ 1000
- 位置计算准确率 ≥ 99%

### 3.2 安全要求
- 用户密码加密存储（BCrypt）
- 接口访问 token 认证
- 防止位置伪造检测
- SQL 注入防护

### 3.3 可靠性
- 系统可用性 ≥ 99.9%
- 数据备份机制
- 异常自动重试

### 3.4 可扩展性
- 支持多企业/多租户
- 支持打卡点动态配置
- 支持插件化扩展

---

## 4. 技术栈

### 4.1 前端
- Vue 3 + TypeScript
- Vite 构建工具
- Element Plus UI 组件库
- Axios HTTP 客户端
- Vue Router 路由管理
- Pinia 状态管理

### 4.2 后端
- Spring Boot 3.x
- MyBatis-Plus 3.x
- MySQL 8.0
- Redis 7.x
- JWT 认证
- Lombok

### 4.3 部署
- Docker 容器化
- Nginx 反向代理
