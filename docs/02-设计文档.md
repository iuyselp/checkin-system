## 4. 核心算法设计

### 4.1 距离计算（Haversine 公式）
```java
/**
 * 计算两点之间的距离（单位：米）
 * @param lat1 点 1 纬度
 * @param lng1 点 1 经度
 * @param lat2 点 2 纬度
 * @param lng2 点 2 经度
 * @return 距离（米）
 */
public double calculateDistance(double lat1, double lng1, double lat2, double lng2) {
    double earthRadius = 6371000; // 地球半径（米）
    double dLat = Math.toRadians(lat2 - lat1);
    double dLng = Math.toRadians(lng2 - lng1);
    double a = Math.sin(dLat / 2) * Math.sin(dLat / 2) +
               Math.cos(Math.toRadians(lat1)) * Math.cos(Math.toRadians(lat2)) *
               Math.sin(dLng / 2) * Math.sin(dLng / 2);
    double c = 2 * Math.atan2(Math.sqrt(a), Math.sqrt(1 - a));
    return earthRadius * c;
}
```

### 4.2 打卡状态判断
```java
/**
 * 判断打卡状态
 * @param checkTime 打卡时间
 * @param rule 考勤规则
 * @return 打卡状态
 */
public CheckStatus determineStatus(LocalTime checkTime, AttendRule rule) {
    LocalTime workStart = rule.getWorkStartTime();
    LocalTime workEnd = rule.getWorkEndTime();
    int lateGrace = rule.getLateGraceMinutes();
    
    if (checkTime.isBefore(workStart.plusMinutes(lateGrace))) {
        return CheckStatus.NORMAL; // 正常
    } else if (checkTime.isBefore(workStart.plusMinutes(30))) {
        return CheckStatus.LATE; // 迟到
    } else {
        return CheckStatus.ABSENT; // 缺勤
    }
}
```

## 5. Redis 缓存设计

### 5.1 缓存键设计
```
# 用户 Token
user:token:{userId} -> JWT Token (TTL: 2h)

# 用户信息
user:info:{userId} -> User JSON (TTL: 30m)

# 打卡点信息
point:info:{pointId} -> CheckPoint JSON (TTL: 1h)

# 当日打卡状态
checkin:today:{userId}:{date} -> CheckStatus (TTL: 24h)

# 接口限流
rate:limit:{api}:{userId} -> count (TTL: 1min)
```

### 5.2 缓存策略
- 热点数据：用户信息、打卡点信息
- 会话数据：Token、临时状态
- 统计数据：每日凌晨预计算
